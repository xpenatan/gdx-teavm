# TeaVM Desktop C Build Instructions

This guide explains how to compile Java code to C using TeaVM and build a native executable with CMake.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Java 21** (or compatible version)
  - Set the correct Java version in Project Settings -> Gradle JVM
  ![img.png](images/img.png)
  
- **Visual Studio Community 2022** (or later) with C++ development tools on Windows
  - Older versions may work but have not been tested
  
- **CMake** (required for building the C code)
- **Linux/macOS native dependencies**
  - A C compiler toolchain
  - GLFW, OpenGL, and GLEW development packages discoverable by CMake

## Step 1: Generate and Build

Run a Gradle task to generate C code using TeaVM and build the native executable:

**Debug build**
```
gradlew :examples:basic:desktop-c:basic_desktop_c_debug_build
```

**Release build**
```
gradlew :examples:basic:desktop-c:basic_desktop_c_release_build
```

The launcher always uses `TeaVMOptimizationLevel.FULL`.

**Generate C code only**
```
gradlew :examples:basic:desktop-c:basic_desktop_c_generate
```

The short build task is a debug-build alias:
```
gradlew :examples:basic:desktop-c:basic_desktop_c_build
```

## Step 2: Run

Run the debug executable through Gradle:

```
gradlew :examples:basic:desktop-c:basic_desktop_c_debug_run
```

You can also choose the configuration explicitly:

```
gradlew :examples:basic:desktop-c:basic_desktop_c_debug_run
gradlew :examples:basic:desktop-c:basic_desktop_c_release_run
gradlew :examples:basic:desktop-c:basic_desktop_c_release_console_run
```

The run tasks generate C code, compile the native executable, and start it. On Windows, debug run and release console run open a console window and run the native executable inside it, so stdout/stderr logs are visible while the app is running.

The Gradle tasks delegate native build/run behavior to ```TeaGLFWBackend```. A Java launcher can configure the same behavior directly:

```java
TeaGLFWBackend backend = new TeaGLFWBackend()
        .setBuildType(TeaGLFWBackend.NativeBuildType.DEBUG)
        .setBuildExecutableAfterBuild(true)
        .setRunExecutableAfterBuild(true);
```

The example Gradle tasks pass normal launcher arguments:

```text
Debug
Debug build
Debug run console
Release build
Release run
Release run console
```

## Manual Native Build

You can still compile the generated C code manually.

### Option A: Using Automated Scripts (Recommended)

Build scripts are automatically generated to simplify the build process:
- ```app_debug.bat``` - Builds a debug executable
- ```app_release.bat``` - Builds an optimized release executable
- ```app_debug.sh``` - Builds a debug executable on Linux/macOS
- ```app_release.sh``` - Builds an optimized release executable on Linux/macOS

Run the script for your platform. It will:
1. Run CMake to configure the project
2. Use the native build tool generated by CMake
3. Compile the C code into an executable

### Option B: Manual Build with CMake

1. Open a terminal in the ```build/dist``` folder
2. Run CMake to generate build files:
   ```
   cmake .
   ```
3. Open the generated Visual Studio solution file
4. Build the project in Visual Studio

### Linux/macOS

Just type

```shell
cmake .
make
```

or to build debug version, following:

```shell
cmake -DCMAKE_BUILD_TYPE=Debug
make
```

## Output

After successful compilation, you will find:
- The executable in ```build/dist/c/release/```
- The assets folder in the same directory

## Customization

The build scripts and CMakeLists.txt files are generated by the ```TeaGLFWBackend``` class from templates in ```backends/backend-glfw/src/main/resources/templates/glfw```. You can modify those templates to customize the build configuration as needed.
